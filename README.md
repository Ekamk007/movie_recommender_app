# 🎬 CineMatch — Movie Recommendation Engine

A Streamlit web app that recommends movies using **content-based filtering + sentiment analysis**, powered by the TMDB 5000 dataset and The Movie Database API.

---

## 🚀 Deploy to Streamlit Community Cloud (free)

1. Fork this repo on GitHub
2. Run the notebook locally first to generate the pickle files (see below)
3. Commit `movies.pkl` and `similarity.pkl` to the repo (or use Git LFS)
4. Go to [share.streamlit.io](https://share.streamlit.io) → New app → pick repo → `app.py` → Deploy

## ⚙️ Local Setup

### Step 1 — Install dependencies
```bash
pip install -r requirements.txt
# For the notebook additionally:
pip install transformers torch pmdarima jupyter
```

### Step 2 — Get the TMDB 5000 dataset
Download from Kaggle: [TMDB 5000 Movie Dataset](https://www.kaggle.com/datasets/tmdb/tmdb-movie-metadata)

Place in the project folder:
- `tmdb_5000_movies.csv`
- `tmdb_5000_credits.csv`

### Step 3 — Run the notebook to build pickle files
```bash
jupyter notebook Movie.ipynb
# Run all cells → generates movies.pkl + similarity.pkl
```

### Step 4 — Launch the app
```bash
streamlit run app.py
```

## 🧠 How It Works

### 1. Feature Engineering (Notebook)
Each movie's tags are built by concatenating:
- Plot overview (word-tokenised)
- Genres (space-collapsed)
- Keywords
- Top 3 cast members
- Director

All tokens are lower-cased and Porter-stemmed.

### 2. Vectorisation
Bag-of-Words with `CountVectorizer` (max 5 000 features, English stop-words removed) → `(4809, 5000)` sparse matrix.

### 3. Similarity
Pairwise cosine similarity → `(4809, 4809)` dense matrix, serialised to `similarity.pkl`.

### 4. Hybrid Scoring
```
final_score = 0.8 × cosine_similarity + 0.2 × sentiment_score
```
- **App**: keyword-based sentiment (fast, no GPU needed)
- **Notebook**: HuggingFace `DistilBERT` transformer pipeline (full accuracy)

### 5. TMDB API
Live posters, overviews, ratings, runtimes, and user reviews fetched at query time (cached 1 h).
